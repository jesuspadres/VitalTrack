# VitalTrack

A serverless, HIPAA-aware health biomarker tracking platform with AI-powered insights. Built entirely on AWS.

Users submit blood panel results (manually or via CSV upload), visualize biomarker trends over time, and receive personalized health insights generated by Amazon Bedrock (Claude).

## Architecture

```
                           CloudFront (CDN)
                                │
                          ┌─────┴─────┐
                          │  S3 (SPA) │
                          └───────────┘

    Client ──▶ API Gateway (Cognito JWT) ──▶ Lambda (Python 3.12)
                                                    │
                        ┌───────────────────────────┼──────────────────┐
                        ▼                           ▼                  ▼
                   DynamoDB                    S3 (uploads)       EventBridge
                (biomarkers,                       │                   │
              insights, audit)                     ▼                   ▼
                                            Lambda (CSV parser)  Step Functions
                                                   │              (Express)
                                                   ▼                   │
                                             EventBridge          ┌────┼────┐
                                                                  ▼    ▼    ▼
                                                            Bedrock  DynamoDB SNS
                                                           (Claude)
```

### AWS Services (13+)

| Layer | Service | Purpose |
|-------|---------|---------|
| Frontend | S3 + CloudFront | Static SPA hosting with OAC, TLS 1.2+ |
| Auth | Cognito User Pools | Registration, login, JWT tokens |
| API | API Gateway (REST) | HTTPS endpoints with Cognito authorizer |
| Compute | Lambda (Python 3.12) | 8 functions, ARM64, X-Ray traced |
| Database | DynamoDB | 3 tables — biomarkers, insights, audit log |
| Storage | S3 | CSV uploads with presigned URLs |
| Events | EventBridge | Custom event bus for async workflows |
| Orchestration | Step Functions | Express workflow for insight pipeline |
| AI | Bedrock (Claude 3.5 Sonnet) | Health insight generation |
| Notifications | SNS | Email alerts when insights are ready |
| Queue | SQS | Dead-letter queue for failed events |
| Observability | CloudWatch + X-Ray | Dashboard, alarms, distributed tracing |
| IaC | CDK (TypeScript) | 7 stacks, cdk-nag security checks |

## Getting Started

### 1. Install Dependencies

```bash
make install
```

Or individually:

```bash
# Backend
cd backend && pip install -r requirements.txt -r requirements-dev.txt

# Infrastructure
cd infrastructure && npm ci

# Frontend
cd frontend && npm ci
```

### 2. Configure Environment Variables

Copy the example files and fill in your values:

```bash
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env.local
```

See each `.env.example` for required variables and descriptions.

### 3. Run Tests

```bash
# All tests
make test

# Backend only (75+ unit tests with moto AWS mocking)
make test-backend

# Frontend
make test-frontend

# CDK snapshot tests
make test-infra
```

### 4. Lint & Type Check

```bash
make lint
```

### 5. Synthesize CloudFormation

```bash
make synth
```

### 6. Deploy (Dev)

```bash
make deploy-dev
```

This deploys all 7 stacks to your AWS account with the `dev` stage configuration.

### 7. Run Frontend Locally

```bash
cd frontend && npm run dev
```

The app runs at `http://localhost:5173`. Requires the backend to be deployed (API Gateway URL in `.env.local`).

## Data Flow

### Biomarker CRUD (synchronous)
Client -> API Gateway (JWT auth) -> Lambda -> DynamoDB -> Response

### CSV Upload (asynchronous)
Client -> S3 presigned URL -> S3 -> EventBridge -> Lambda (parser) -> DynamoDB -> EventBridge (`BiomarkersIngested`)

### AI Insight Generation (event-driven)
EventBridge (`BiomarkersIngested`) -> Step Functions -> Fetch History -> Bedrock (Claude) -> Store Insight -> SNS Notify

## Security

- **Encryption at rest**: DynamoDB (AWS-owned), S3 (SSE-S3), SQS (SSE)
- **Encryption in transit**: TLS 1.2+ enforced everywhere
- **Auth**: Cognito JWT with per-request user isolation
- **Data isolation**: Every DynamoDB query includes `userId` from JWT — no cross-tenant access
- **Input validation**: Pydantic v2 strict mode, CSV sanitization (formula injection prevention)
- **IaC security**: cdk-nag AWS Solutions checks on all 7 stacks
- **CI security gates**: Bandit, pip-audit, npm audit

## Observability

- **CloudWatch Dashboard**: API Gateway metrics, Lambda errors/duration, DynamoDB throttles, Step Functions, DLQ depth
- **Alarms**: API 5xx spike, Lambda errors, DynamoDB throttle, SFN failures, DLQ non-empty
- **X-Ray**: Distributed tracing across all Lambda functions
- **Structured logging**: JSON logs with correlation IDs

## CI Pipeline

GitHub Actions runs on every push:

1. **Backend**: Ruff lint + format, MyPy strict, Bandit, pip-audit, pytest (80% coverage gate)
2. **Infrastructure**: CDK synth, Jest snapshot tests, npm audit
3. **Frontend**: TypeScript strict check, Vitest, npm audit
